# Database Migrations

These SQL files were generated by `drizzle-kit generate` during early development.

## Current workflow

This project uses **`db:push`** (schema push), not migrations:

```bash
bun run db:push
```

`db:push` diffs the Drizzle schema against the live database and applies changes
directly. It is the recommended workflow for development and is safe as long as
destructive changes (column drops, renames) are handled with care.

## Why the SQL files are here

The `.sql` files and `meta/` directory are Drizzle's generated migration artifacts.
They are kept for reference but are **not applied** by the current deployment process.
Do not run `drizzle-kit migrate` unless intentionally switching to a migration-based
workflow.

## Switching to migrations (production recommendation)

If you want stricter control over schema changes in production:

1. Delete or archive these files
2. Run `bun run db:generate` to create a fresh baseline migration
3. Change the deployment step to `bun run db:migrate` instead of `bun run db:push`
4. Document each schema change with a new migration before deploying
