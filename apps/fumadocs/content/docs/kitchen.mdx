## Kitchen Display System (KDS)

**Navigate to:** Dashboard → Kitchen Display

The Kitchen Display shows all active orders as tickets in real time.

### Layout

Each ticket shows: order number, time received, all items with special notes, and modifiers as grey chips.

### Department Filter

Use the filter tabs at the top to view only orders for a specific department (e.g., "Meals", "Pastry", "Beverages").

### Colour Coding

| Colour | Meaning |
|--------|---------|
| Grey/White | Fresh order |
| Yellow | Waiting longer than expected |
| Red | Too long — needs immediate attention |

### Marking Items

Click an item to mark it prepared (dims/checks off). When all items done, click **Mark Ready** — ticket turns green.

### Course-Based Ordering

For multi-course meals, items are grouped by course. Kitchen staff can fire items by course.

### Auto-Refresh

The KDS refreshes automatically every few seconds.

## Tables

**Navigate to:** Dashboard → Tables

### Table Status

| Status | Colour | Meaning |
|--------|--------|---------|
| Available | Green | Ready for seating |
| Occupied | Red | Active order in progress |
| Reserved | Blue | Booked for upcoming reservation |

### Managing Tables

- Assign orders to tables when creating Dine In orders
- Move orders between tables
- Clear tables after service
- Configure tables in Settings → Tables

See [Floor Plans & Table Layout](/docs/floor-plan) for visual table management.
